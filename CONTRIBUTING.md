# 贡献指南

感谢你对 video-creator 的兴趣！这是一个纯文本 Markdown 项目，贡献门槛极低。

## 三种贡献方式

### 1. 提 Issue（零门槛）

遇到以下情况直接开 Issue：

- 使用中发现的 bug 或异常
- 想新增的功能或改进建议
- 某个阶段的产出格式不稳定
- 对评分标准的疑问
- 适配新平台的建议

不需要任何编程知识，描述清楚问题即可。

### 2. 小修小补（会 Markdown 即可）

适合改一行到十几行的小改动：

1. Fork 本仓库
2. 直接编辑 `SKILL.md` 或 `README.md`
3. 提交 PR，描述改了什么、为什么改

常见的小改动类型：
- 格式常量 F1-F5 的微调
- 文档错别字或措辞优化
- 时长匹配表的数据修正
- 合规审查清单的条目增补

### 3. 深度改造（建议先开 Issue 讨论）

适合新增阶段、重构流水线、适配新平台等大改动：

1. 先开 Issue 描述你的想法
2. 讨论确认方向后再动手
3. 改动后完整走一遍 8 步流水线验证

## 文件说明

| 文件 | 性质 | 说明 |
|------|------|------|
| `SKILL.md` | 核心文件 | 本项目的全部逻辑，约 49KB 纯文本 |
| `meta.json` | 元数据 | 名称、版本、用例等元信息 |
| `README.md` | 项目文档 | 不参与 Skill 执行 |
| `CONTRIBUTING.md` | 本文件 | 贡献规范 |

> **只改核心文件**：`SKILL.md` 是唯一生效的主文件，其余均为辅助文件。

## SKILL.md 结构速览

```
---
frontmatter（名称/描述/触发词）
---
```

## PR 规范

- 标题：[类型] 简短描述，如 `[fix] 修正阶段3分镜表格式`、`[feat] 新增爱奇艺平台合规规则`
- 描述：改了什么 + 为什么改 + 验证结果
- 一个 PR 只做一件事

## 行为准则

- 友善讨论，对事不对人
- 尊重不同的创作审美和意见
- 欢迎新手，不设技术门槛
