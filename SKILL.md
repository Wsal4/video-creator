# AI视频全流程创作技能 v4.0

## 架构总览

**三层架构**：Main Agent 调度 → file-agent 执行 → Python 脚本验证

```mermaid
graph LR
    S0[阶段0 选题配置] --> C1{确认1} --> S1[阶段1 故事正文] --> C2{确认2} --> S2[阶段2 脚本表] --> S3[阶段3 分镜表] --> S35[阶段3.5 段落故事板] --> S4[阶段4 提示词包] --> S5[阶段5 合规+质量] --> S6[阶段6 产出汇总]
```

**关键设计原则**：

| 原则 | 说明 |
|------|------|
| **隔离上下文** | 每阶段独立 `dispatch_task`，不累积上下文压力。长内容自动拆章逐章独立调用 |
| **硬约束验证** | 每阶段后 Python 脚本检查体量/完整性，不达标阻断而非靠 LLM 自觉 |
| **文件锚点** | 阶段间传递文件路径，下一阶段 `read_file` 读取完整内容，不压缩摘要 |
| **独立审查** | 阶段5质量审查用全新 file-agent 调用，不继承创作上下文，只读产出物文件 |
| **可验证反馈** | 对话展示验证结果（行数/分镜数/通过状态），而非不可验证的摘要数字 |

## 时长-叙事结构匹配表

| 时长区间 | 叙事结构层级 | 故事最低字数 | 脚本最低行数 | 分镜最低数量 | 分镜单镜上限 | 最低障碍升级次数 |
|----------|------------|------------|------------|------------|------------|----------------|
| ≤ 1分钟 | 单一事件，1反转 | ≥ 500字 | ≥ 8行 | ≥ 6个 | 30秒 | ≥ 1次 |
| 1-5分钟 | 三幕（简化），1反转，A/B线 | ≥ 1,500字 | ≥ 15行 | ≥ 12个 | 30秒 | ≥ 1次 |
| 5-15分钟 | 三幕（完整），2反转，A/B+支线 | ≥ 4,000字 | ≥ 30行 | ≥ 25个 | 30秒 | ≥ 2次 |
| 15-30分钟 | 三幕（深描），3反转，2条人物弧光 | ≥ 8,000字 | ≥ 55行 | ≥ 45个 | 30秒 | ≥ 2次 |

## 【格式常量定义】

> **核心原则**：以下四个产出物 + 一个衍生格式的模板在本文档中**唯一定义于此**。后续各阶段必须严格引用本常量，严禁在任何步骤中自行修改列数、列名、字段名或输出格式。无论视频时长长短，格式恒定不变。

### F1. 故事正文格式（8项必含标注）

```
①【期待钩子：XXX】— 开头3句话内建立悬念
②【障碍升级：XXX（位置：第X段）】— 逐次升级
③【反转：XXX（回收伏笔：第X段）】— 至少1个真正转折
④【期待闭环：回应开头XXX】— 结局回应开头悬念
⑤【彩蛋：XXX】— 至少3个，前中后分布
⑥【伏笔：XXX（回收位置：第X段）】— 至少2个，结尾回收
⑦ 首次出场人物加粗姓名
⑧ 角色、场景、道具、彩蛋用【】标注
```

### F2. 故事脚本格式（9列，固定不可变）

```
| 序号 | 画面内容 | 时长 | 人物 | 台词 | 动作 | 镜头（自动匹配） | 旁白 | 备注 |
```

列数：**9列**。列名逐字固定，禁止增减、重命名或调换顺序。

### F3. 分镜表格式（13列，固定不可变）

```
| 分镜号 | 景别 | 角度 | 构图 | 镜头描述 | 时长 | 画面内容 | 台词/音色 | 音效 | 光源 | 节奏标记 | 彩蛋提示 | 视频生成提示词 |
```

列数：**13列**。列名逐字固定，禁止增减、重命名或调换顺序。

### F4. 视频生成提示词格式（8字段，固定不可变）

```
【主体描述】{核心人物/物体的外观、姿态、位置}
【动作/细节】{关键动作、面部表情、道具状态、特效元素}
【场景环境】{地点、时间、天气}
【光影色调】{光源方向+强度、色调、氛围}
【画风/风格】{阶段0确认的风格词}
【画质参数】{分辨率、清晰度等}
【镜头运镜】{景别、运镜方式——仅Seedance视频生成时使用}
【约束条件】{排除项，如"避免抖动""避免面部变形""避免模糊"}
```

字段数：**8字段**。字段名和顺序逐字固定，禁止增减、重命名或调换顺序。

### F5. 15秒段落故事板格式（固定模板）

**拆分规则**：
- 每个段落总时长 **≤15秒**
- 段落总数公式：`N = ceil(全片总秒数 ÷ 15)`
- 同一段落内为同一场景，禁止跨场景合并
- 优先按场景切换点自然拆分，场景内超15秒按动作节点二次拆分

**段落输出模板**：

```
## P{N}：{段落名}（{总时长}s）

**画面风格和类型**：{CG动画/写实/二次元等}，{风格描述}，{色调}。

生成一个由以下{N}个分镜组成的视频，总时长{总时长}s。

【衔接：从P{N-1}的{某某画面}叠化至本段{某某画面}】{P1省略此行}

**本片段场景设定在**：{场景氛围描述}的{{{{场景名称}}场景图}}，{场景补充描述}。

**角色**：{{{{角色名}}全身图}}是{角色名}{多个角色逐行列出}。

**道具**：{{{{道具名}}}}是{道具名}{无道具段落可省略}。

**分镜1，{X}s**：{画面描述}。台词："{台词内容}"（音色：{性别}，{年龄区间}音色，{质感}，{发音方式}，{气息}，{吐字}）；无台词则写"无台词。"音效：{音效描述}。

**分镜2，{X}s**：...
```

**占位符规范**：
- 场景：`{{{{场景名称}}场景图}}`
- 角色：`{{{{角色名}}全身图}}`
- 道具：`{{{{道具名}}}}`

> ⛔ **格式保全红线（最高优先级）**：
> 段落故事板是可直接投喂给 AI 视频生成工具的最终产出物。以下行为在任何阶段（包括阶段 6 产出汇总）均被**绝对禁止**：
> 1. **禁止表格化**——不得将段落故事板转换为 Markdown/Word 表格（如"帧数预算 | 场景摘要 | 视觉叙事"等列）
> 2. **禁止摘要化**——不得将逐分镜描述压缩为"场景摘要""视觉叙事"等概括性文字
> 3. **禁止丢失占位符**——`{{场景}}` `{{角色}}` `{{道具}}` 双花括号引用必须原样保留
> 4. **禁止丢失台词音色标注**——台词后的`（音色：…）`标注不得删除或简化
> 5. **禁止丢失音效描述**——每个分镜末尾的`音效：…`不得删除
> 6. **禁止丢失衔接标注**——P2 起的`【衔接：…】`不得删除
> 7. **禁止重构**——若 file-agent 在阶段 6 无法读取到阶段 3.5 的原始 `.docx` 文件内容，**不得自行编造替代内容**，必须向 Main Agent 报告缺失并要求提供原文文件路径
>
> 违规判定：以上 7 条任一违反，阶段 5 格式一致性验证（F5验证）判 ❌ 阻断。

**完整性验证**（生成后强制执行）：
1. 段落总数 × 15 ≥ 全片总时长
2. 所有分镜号已按顺序分配到对应段落（无遗漏、无重复）
3. 每个段落衔接标注完整（P1除外）
4. 每个段落总时长 ≤ 15秒

  逐章独立生成，每章保存为独立文件 `故事正文_第N章.docx`。

  ## 验证脚本（必须执行）

  ```python
  import os, sys
  from docx import Document

  MIN_WORDS = {故事最低字数}
  MIN_CHAPTERS = {最少章数}
  STORY_DIR = r"{故事正文保存目录}"

  def count_words(path):
      doc = Document(path)
      return sum(len(p.text) for p in doc.paragraphs)

  files = sorted([f for f in os.listdir(STORY_DIR) if '故事正文' in f and f.endswith('.docx')])
  total = 0
  for f in files:
      wc = count_words(os.path.join(STORY_DIR, f))
      total += wc
      print(f"{'✅' if wc > 0 else '❌'} {f}: {wc} 字")

  word_ok = total >= MIN_WORDS * 0.95
  ch_ok = len(files) >= MIN_CHAPTERS
  print(f"\n总字数: {total} / 下限: {MIN_WORDS} (容差: {int(MIN_WORDS*0.95)}) → {'✅' if word_ok else '❌ 不达标'}")
  print(f"章节数: {len(files)} / 下限: {MIN_CHAPTERS} → {'✅' if ch_ok else '❌ 不达标'}")
  print(f"\n最终判定: {'✅ 通过' if (word_ok and ch_ok) else '❌ 阻断，重做阶段1'}")

  if not (word_ok and ch_ok):
      sys.exit(1)
  ```

  完整故事正文仅在 Word 文档中，不在对话中输出。
  </current_task>
```

# 阶段 2：脚本表

## 派发模板

```
agent_name: file-agent
task:
  <overall_goal>用户要求基于「{题目}」创作{时长}的{风格}风格视频全套物料。阶段2目标：基于故事正文生成标准化脚本表。</overall_goal>
  <current_task>
  基于故事正文和选题设定生成脚本表。

  ## 输入文件
  [选题设定.docx](<路径>)
  [故事正文.docx](<路径>)  {或逐章文件路径}

  ## 硬约束
  - 脚本总行数 ≥ {脚本最低行数}（±5%容差）
  - 全表时长列加总 = 总秒数 {总秒数}（误差 ±5秒）
  - 单镜时长 ≤ {分镜单镜上限}，超限自动拆分

  ## 【格式强制】表格必须使用 F2 定义的9列格式，严禁增减列数或修改列名。

  | 序号 | 画面内容 | 时长 | 人物 | 台词 | 动作 | 镜头（自动匹配） | 旁白 | 备注 |

  **画面内容写作规范**：
  - 禁止写地点（地点统一写在备注列开头）
  - 只写能被镜头拍出来的视觉内容（动作、道具状态、光影）
  - 彩蛋/伏笔在画面内容和备注列重复标注
  - 镜头列根据情绪/节奏/画面信息/风格自动选择景别/运动/角度/效果

  ## 分章规则
  {时长>10分钟：逐章独立生成。每章读对应故事正文章节，输出 `脚本表_第N章.docx`。}
  {时长≤10分钟：单次生成完整脚本表 `脚本表.docx`。}

  ## 自动验证（生成过程中）
  - 画面可行性检查（AI 可生成，无逻辑矛盾）
  - 分镜超限自动拆分
  - 时长列加总校验

  ## 验证脚本（生成完成后必须执行）

  ```python
  import os, sys
  from docx import Document

  MIN_ROWS = {脚本最低行数}
  TARGET_SEC = {总秒数}
  SCRIPT_DIR = r"{脚本表保存目录}"

  def count_rows(path):
      doc = Document(path)
      return sum(1 for t in doc.tables for row in t.rows if row.cells[0].text.strip().isdigit())

  def sum_duration(path):
      doc = Document(path)
      total = 0
      for t in doc.tables:
          for row in t.rows:
              cells = row.cells
              if cells[0].text.strip().isdigit() and len(cells) >= 3:
                  try: total += float(cells[2].text.strip().replace('s',''))
                  except: pass
      return total

  from docx import Document

  STORYBOARD_DIR = r"{段落故事板保存目录}"

  def count_shots(path):
      doc = Document(path)
      count = 0
      for t in doc.tables:
          for row in t.rows:
              if row.cells[0].text.strip().isdigit():
                  count += 1
      return count

  import re

  def analyze_storyboard(path):
      with open(path, 'r', encoding='utf-8') as f:
          text = f.read()

      s_count = len(shots)
      # 提取每个段落时长
      durations = re.findall(r'## P\d+：.+?（(\d+)s）', text)
      total_dur = sum(int(d) for d in durations)
      # 检查是否有衔接标注（P2起）

  sb_files = sorted([f for f in os.listdir(STORYBOARD_DIR) if '段落故事板' in f and f.endswith('.docx')])
  total_paragraphs = 0
  total_sb_shots = 0
  all_shot_ids = set()

  for f in sb_files:
      path = os.path.join(STORYBOARD_DIR, f)
      pc, sc, td, hl, sids = analyze_storyboard(path)
      total_paragraphs += pc
      total_sb_shots += sc
      all_shot_ids |= sids
      print(f"{'✅' if pc > 0 else '❌'} {f}: {pc} 段落, {sc} 分镜, 覆盖时长 {td}s, 衔接{'完整' if hl else '缺失'}")

  print(f"实际段落数: {total_paragraphs} → 覆盖上限 {total_paragraphs * 15}s / 目标 {TARGET_SEC}s → {'✅' if cov_ok else '❌ 覆盖不足'}")

  print(f"\n最终判定: {'✅ 通过' if (cov_ok and shot_ok) else '❌ 阻断，重做阶段3.5'}")

  if not (cov_ok and shot_ok):
